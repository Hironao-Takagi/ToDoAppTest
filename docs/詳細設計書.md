## 1. システム概要

### 1-1. 目的
タスクの追加・完了・削除を中心とした日本語対応のToDoアプリ。
開始予定日・終了予定日・実績日の管理、繰り返しタスク設定、カレンダービューによる可視化を提供する。

### 1-2. 主な機能一覧

| # | 機能 | 概要 |
|---|------|------|
| 1 | タスク追加 | タスク名・開始予定日・終了予定日・繰り返し設定を入力して追加 |
| 2 | タスク完了 | 完了時に開始実績・終了実績を入力してから完了処理 |
| 3 | タスク削除 | 個別削除・完了済み一括削除 |
| 4 | 日付編集 | 追加後に4つの日付フィールドを編集・保存 |
| 5 | フィルター | すべて／未完了／完了済みで一覧を絞り込み |
| 6 | 繰り返し設定 | なし／毎日／平日のみ／毎週（曜日指定）／毎月／毎年 |
| 7 | カレンダービュー | 月表示カレンダーにタスクの期間を可視化 |
| 8 | データ永続化 | ブラウザの localStorage にデータを自動保存 |

## 4. データモデル

### 4-1. Todo型

```typescript
type Todo = {
  id: string;           // 一意ID（タイムスタンプ + ランダム文字列）
  text: string;         // タスク名
  completed: boolean;   // 完了フラグ
  createdAt: number;    // 作成日時（Unix timestamp ms）
  plannedStart: string; // 開始予定日（YYYY-MM-DD形式）
  plannedEnd: string;   // 終了予定日（YYYY-MM-DD形式）
  actualStart: string;  // 開始実績日（YYYY-MM-DD形式）
  actualEnd: string;    // 終了実績日（YYYY-MM-DD形式）
  recurrence: Recurrence; // 繰り返し設定
};
```

### 4-2. Recurrence型

```typescript
type RecurrenceType = "none" | "daily" | "weekdays" | "weekly" | "monthly" | "yearly";

type Recurrence = {
  type: RecurrenceType; // 繰り返し種別
  weekDays?: number[];  // 曜日指定（weekly時のみ使用）0=日曜〜6=土曜
};
```

### 4-3. 補助型

```typescript
type Filter = "all" | "active" | "completed"; // リストフィルター
type View   = "list" | "calendar";             // 表示モード
```

### 4-4. localStorage 保存形式

- **キー**: `todos-v2`
- **値**: `Todo[]` を `JSON.stringify` した文字列
- **マイグレーション**: 旧データ（`recurrence`フィールドなし）は読み込み時に `{ type: "none" }` を補完

## 5. 状態管理

`page.tsx` 内の `Home` コンポーネントが `useState` で全状態を管理する（単一コンポーネント構成）。

### 5-1. 状態一覧

| 状態変数 | 型 | 初期値 | 用途 |
|----------|----|--------|------|
| `todos` | `Todo[]` | `[]` | タスクデータ本体 |
| `mounted` | `boolean` | `false` | SSRハイドレーション制御 |
| `input` | `string` | `""` | 追加フォームのタスク名入力値 |
| `addPlannedStart` | `string` | `""` | 追加フォームの開始予定日 |
| `addPlannedEnd` | `string` | `""` | 追加フォームの終了予定日 |
| `addRecurrence` | `Recurrence` | `{ type: "none" }` | 追加フォームの繰り返し設定 |
| `formErrors` | `object` | `{}` | 追加フォームのバリデーションエラー |
| `filter` | `Filter` | `"all"` | リストの絞り込み条件 |
| `expandedIds` | `Set<string>` | `new Set()` | 展開中タスクのID集合 |
| `editDates` | `Record<string, Partial<Todo>>` | `{}` | 日付編集パネルの一時編集値 |
| `completingId` | `string \| null` | `null` | 完了処理中のタスクID |
| `completionDates` | `object` | `{ actualStart: "", actualEnd: "" }` | 完了パネルの実績日入力値 |
| `completionErrors` | `object` | `{ actualStart: false, actualEnd: false }` | 完了パネルのバリデーションエラー |
| `view` | `View` | `"list"` | 表示モード（リスト/カレンダー） |
| `selectedDay` | `string \| null` | `null` | カレンダーで選択中の日付 |
| `calYear` | `number` | 現在年 | カレンダー表示年 |
| `calMonth` | `number` | 現在月 | カレンダー表示月（0始まり） |

### 5-2. 副作用（useEffect）

| タイミング | 処理内容 |
|-----------|---------|
| 初回マウント時 | localStorage から `todos-v2` を読み込み、`mounted = true` にセット |
| `todos` または `mounted` 変化時 | `mounted === true` の場合のみ `todos-v2` に保存 |

※ `mounted` フラグによりサーバーサイドレンダリング時の誤書き込みを防止する。

## 6. 機能詳細

### 6-1. タスク追加

**バリデーション条件**: 以下3項目すべて必須
- タスク名（空白のみ不可）
- 開始予定日
- 終了予定日

**ID生成方式**:
```
id = `${Date.now()}-${Math.random().toString(36).slice(2)}`
```
タイムスタンプ（ms）とランダム文字列の組み合わせで一意性を確保。

**追加後の処理**: 入力欄をクリアし、タスク名入力欄にフォーカスを戻す。

---

### 6-2. タスク完了フロー

完了ボタン（丸ボタン）クリック時、即時完了ではなく以下のフローを取る。

```
[完了ボタン押下]
    ↓
[完了パネルをインライン表示]
  ・開始実績日（デフォルト：今日）
  ・終了実績日（デフォルト：今日）
    ↓
[バリデーション]（両方必須）
    ↓
[completedフラグをtrueに更新]
[actualStart / actualEnd を保存]
    ↓
[繰り返し設定がある場合]
  → createNextInstance() で次インスタンスを生成し todos に追加
    ↓
[完了パネルを閉じる]
```

**未完了に戻す場合**: `completed = false`、`actualStart`・`actualEnd` を空文字にリセット。

---

### 6-3. 繰り返しタスク

#### 繰り返し種別と挙動

| 種別 | 値 | カレンダー表示条件 | 次インスタンスの開始日 |
|------|----|--------------------|------------------------|
| なし | `none` | plannedStart 〜 plannedEnd の期間 | - |
| 毎日 | `daily` | plannedStart 以降の全日 | +1日 |
| 平日のみ | `weekdays` | plannedStart 以降の月〜金 | +1日（土日スキップ） |
| 毎週 | `weekly` | 指定曜日（未指定時は開始日の曜日） | +7日 |
| 毎月 | `monthly` | 毎月同日 | +1ヶ月 |
| 毎年 | `yearly` | 毎年同月日 | +1年 |

#### 完了済み繰り返しタスクの表示

完了したインスタンスは繰り返しパターンを無視し、**そのインスタンスの plannedStart 〜 plannedEnd の期間のみ**カレンダーに表示する。

#### 次インスタンス生成ロジック（`createNextInstance`）

1. 現在の `plannedStart` を基準に次の開始日を算出
2. `plannedEnd` が設定されている場合、元の期間（日数）を維持して次の終了日を算出
3. 新しい `id`・`createdAt` を付与し、`completed`・`actualStart`・`actualEnd` をリセット

---

### 6-4. カレンダービュー

#### 表示ロジック（`occursOnDay` 関数）

各日付セルに対し、全タスクをフィルタリングして表示対象を決定する。

```
入力: Todo, dayStr(YYYY-MM-DD)

[完了済み かつ 繰り返しタスク]
  → plannedStart 〜 plannedEnd の期間内か判定

[未完了または一回限りタスク]
  → recurrence.type に応じた判定
     none     : plannedStart 〜 plannedEnd の期間内
     daily    : plannedStart 以降
     weekdays : plannedStart 以降 かつ 月〜金
     weekly   : plannedStart 以降 かつ weekDays に含まれる曜日
     monthly  : plannedStart 以降 かつ 同日
     yearly   : plannedStart 以降 かつ 同月日
```

#### カレンダーUI仕様

- 7列グリッド（日〜土）
- 各セルに最大2件のタスクバッジを表示、3件以上は `+N` で省略
- 日付クリックで当日タスクの詳細パネルを下部に表示
- `< >` ボタンで月移動（年またぎ対応）

#### タスクバッジの色分け

| 状態 | 色 |
|------|----|
| 未完了 | インディゴ（`bg-indigo-100 text-indigo-700`） |
| 完了済み | グレー（`bg-stone-100 text-stone-400`） |
| 期限超過（一回限りタスクのみ） | 赤（`bg-red-100 text-red-600`） |

※ 繰り返しタスクは期限超過判定を行わない。

---

### 6-5. 日付編集パネル

タスクカードの `∨` ボタンで展開。4つの日付フィールドを個別に編集可能。

| フィールド | 説明 |
|-----------|------|
| 開始予定日 | タスク開始の予定日 |
| 終了予定日 | タスク終了の予定日 |
| 開始実績 | 実際に着手した日 |
| 終了実績 | 実際に完了した日 |

保存ボタン押下で `todos` を更新し、パネルを閉じる。完了パネルと日付編集パネルは同時に開かない（片方を開くと他方は閉じる）。

---

### 6-6. バリデーション

#### タスク追加フォーム

| フィールド | 必須 | エラーメッセージ |
|-----------|------|-----------------|
| タスク名 | ○ | タスク名を入力してください |
| 開始予定日 | ○ | 選択してください |
| 終了予定日 | ○ | 選択してください |

- エラー発生時は該当フィールドの枠を赤色に変更
- 入力・選択するとそのフィールドのエラーのみ即時解除

#### 完了パネル

| フィールド | 必須 | エラーメッセージ |
|-----------|------|-----------------|
| 開始実績 | ○ | 入力してください |
| 終了実績 | ○ | 入力してください |

## 8. 関数・ロジック詳細

### 8-1. ユーティリティ関数

| 関数名 | 引数 | 戻り値 | 説明 |
|--------|------|--------|------|
| `toYMD(date)` | `Date` | `string` | DateオブジェクトをYYYY-MM-DD形式に変換 |
| `parseYMD(str)` | `string` | `Date \| null` | YYYY-MM-DD文字列をDateオブジェクトに変換 |
| `formatDisplay(str)` | `string` | `string` | YYYY-MM-DDをYYYY/MM/DD形式に変換（表示用） |
| `occursOnDay(todo, dayStr)` | `Todo, string` | `boolean` | 指定日にタスクが表示されるか判定 |
| `createNextInstance(todo)` | `Todo` | `Todo \| null` | 繰り返しタスクの次インスタンスを生成 |
| `getCalendarDays(year, month)` | `number, number` | `(string \| null)[]` | カレンダーグリッド用の日付配列を生成 |
| `recurrenceLabel(r)` | `Recurrence` | `string` | 繰り返し設定の表示ラベルを返す |

### 8-2. イベントハンドラ

| ハンドラ | 処理概要 |
|---------|---------|
| `addTodo()` | バリデーション → Todo追加 → フォームリセット |
| `startCompletion(id)` | 完了パネル表示、デフォルト日付（今日）をセット |
| `confirmCompletion(id)` | バリデーション → 完了更新 → 繰り返し時は次インスタンス追加 |
| `cancelCompletion()` | 完了パネルを閉じる |
| `uncomplete(id)` | 完了フラグをfalseに戻し、実績日をクリア |
| `deleteTodo(id)` | 対象タスクを削除、完了パネルも閉じる |
| `clearCompleted()` | 完了済みタスクをすべて削除 |
| `toggleExpand(id)` | 日付編集パネルの開閉、開く際に編集用状態を初期化 |
| `saveDates(id)` | 編集中の日付をTodoに保存し、パネルを閉じる |
| `prevMonth() / nextMonth()` | カレンダーの月を移動（年またぎ対応） |

## 10. 制約・注意事項

| 項目 | 内容 |
|------|------|
| データ保存場所 | ブラウザの localStorage（サーバー送信なし） |
| データ容量上限 | 約5MB（localStorageの制限） |
| 複数端末共有 | 不可（端末ごとに独立したデータ） |
