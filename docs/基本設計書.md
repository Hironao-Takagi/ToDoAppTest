## 1. システム概要

### 1.1 目的

清掃DXプロジェクトにおける業務タスクを管理するためのWebアプリケーション。定期的な清掃作業を繰り返しタスクとして登録し、予定日・実績日を二重管理することで、業務の進捗把握を支援する。

### 1.2 システム構成図

```
[ブラウザ]
    │
    ├── Next.js (App Router)
    │       └── page.tsx（全機能）
    │
    └── localStorage（データ永続化）
```

### 1.3 技術スタック

| 種別 | 技術 | バージョン |
|------|------|-----------|
| フレームワーク | Next.js | 14.2.5 |
| UIライブラリ | React | ^18 |
| 言語 | TypeScript | ^5 |
| スタイリング | Tailwind CSS | ^3.4.1 |
| ビルドツール | Next.js ビルドシステム | - |
| データ永続化 | localStorage | - |

### 1.4 動作環境

| 項目 | 内容 |
|------|------|
| 実行環境 | クライアントサイド（ブラウザ） |
| 対応ブラウザ | モダンブラウザ（Chrome / Edge / Firefox / Safari） |
| バックエンド | なし（サーバーレス） |
| データベース | なし（localStorageを使用） |

## 2. 機能一覧

### 2.1 機能概要

| 機能ID | 機能名 | 概要 |
|--------|--------|------|
| F-01 | タスク追加 | 新規タスクを登録する |
| F-02 | タスク一覧表示 | タスクをリスト形式で表示する |
| F-03 | タスク完了 | タスクを完了状態にする（実績日入力） |
| F-04 | タスク編集 | タスクの日付情報を編集する |
| F-05 | タスク削除 | タスクを削除する |
| F-06 | 完了済み一括削除 | 完了済みタスクをまとめて削除する |
| F-07 | フィルタリング | タスクを完了状態で絞り込む |
| F-08 | カレンダー表示 | タスクを月間カレンダー形式で表示する |
| F-09 | 繰り返し設定 | タスクに繰り返し周期を設定する |
| F-10 | 期限超過表示 | 期限を超過したタスクを強調表示する |

### 2.2 機能詳細

#### F-01 タスク追加

- タスク名（テキスト）、開始予定日、終了予定日、繰り返し設定を入力して登録
- 全項目必須入力。未入力時はエラー表示
- 登録後、入力フォームをリセット

#### F-03 タスク完了

- 完了ボタン押下時、実績開始日・実績終了日の入力ダイアログを表示
- 両日付を入力して「完了する」ボタン押下で完了状態に遷移
- 繰り返し設定がある場合、次の周期のタスクを自動生成
- 完了済みタスクの完了ボタンを再押下すると未完了に戻す（実績日クリア）

#### F-09 繰り返し設定

| 設定値 | 動作 |
|--------|------|
| なし | 指定期間のみ表示 |
| 毎日 | 開始日以降、毎日表示 |
| 平日のみ | 開始日以降、月〜金のみ表示 |
| 毎週 | 指定曜日に毎週表示（複数曜日選択可） |
| 毎月 | 開始日と同じ日付に毎月表示 |
| 毎年 | 開始日と同じ月日に毎年表示 |

## 4. データ設計

### 4.1 データモデル

#### Todo（タスク）

| フィールド名 | 型 | 必須 | 説明 |
|------------|-----|------|------|
| id | string | ○ | タイムスタンプ＋ランダム文字列 |
| text | string | ○ | タスク名 |
| completed | boolean | ○ | 完了フラグ（初期値: false） |
| createdAt | number | ○ | 作成日時（ミリ秒） |
| plannedStart | string | ○ | 開始予定日（YYYY-MM-DD） |
| plannedEnd | string | ○ | 終了予定日（YYYY-MM-DD） |
| actualStart | string | - | 開始実績日（YYYY-MM-DD） |
| actualEnd | string | - | 終了実績日（YYYY-MM-DD） |
| recurrence | Recurrence | ○ | 繰り返し設定 |

#### Recurrence（繰り返し設定）

| フィールド名 | 型 | 必須 | 説明 |
|------------|-----|------|------|
| type | RecurrenceType | ○ | 繰り返し種別 |
| weekDays | number[] | - | 対象曜日（weekly時のみ, 0=日〜6=土） |

#### RecurrenceType（繰り返し種別）

| 値 | 意味 |
|----|------|
| `"none"` | なし |
| `"daily"` | 毎日 |
| `"weekdays"` | 平日のみ |
| `"weekly"` | 毎週（曜日指定） |
| `"monthly"` | 毎月 |
| `"yearly"` | 毎年 |

### 4.2 データ永続化

| 項目 | 内容 |
|------|------|
| 保存先 | ブラウザ localStorage |
| キー名 | `todos-v2` |
| 形式 | JSON文字列（Todo[] のシリアライズ） |
| 保存タイミング | todos 状態変更時（自動保存） |
| 読込タイミング | ページマウント時 |
| 後方互換性 | 旧データ（recurrenceなし）は自動補完 |

### 4.3 日付フォーマット

| 用途 | 形式 | 例 |
|------|------|----|
| 内部保存 | YYYY-MM-DD | 2026-04-03 |
| 画面表示 | YYYY/M/D | 2026/4/3 |

### 5.2 コンポーネント構成

本アプリは `page.tsx` に全機能を実装したシングルファイル構成。

```
page.tsx
├── 型定義（RecurrenceType, Recurrence, Todo, Filter, View）
├── ヘルパー関数
│   ├── toYMD()          - Date → "YYYY-MM-DD"
│   ├── parseYMD()       - "YYYY-MM-DD" → Date
│   ├── formatDisplay()  - "YYYY-MM-DD" → "YYYY/M/D"
│   ├── occursOnDay()    - タスクの日別表示判定
│   ├── createNextInstance() - 繰り返しタスクの次インスタンス生成
│   └── getCalendarDays() - 月間カレンダー日付配列生成
└── HomePage コンポーネント
    ├── 状態管理（useState × 16）
    ├── データ永続化（useEffect × 2）
    ├── イベントハンドラ
    └── JSX（リストビュー / カレンダービュー）
```

## 6. セキュリティ設計

### 6.1 HTTPセキュリティヘッダ

`next.config.js` にて全ページに以下のヘッダを付与。

| ヘッダ | 値 | 目的 |
|--------|-----|------|
| Content-Security-Policy | `default-src 'self'; ...` | リソース読み込み元の制限、XSS軽減 |
| X-Content-Type-Options | `nosniff` | MIMEタイプスニッフィング防止 |
| X-Frame-Options | `DENY` | クリックジャッキング防止 |
| X-XSS-Protection | `1; mode=block` | ブラウザXSSフィルター有効化 |
| Referrer-Policy | `strict-origin-when-cross-origin` | リファラー情報の制限 |
| Permissions-Policy | `camera=(), microphone=(), geolocation=()` | デバイス機能アクセス禁止 |

### 6.2 XSS対策

- Reactの標準レンダリング（JSX）によりユーザー入力を自動エスケープ
- `innerHTML` / `dangerouslySetInnerHTML` は使用しない

### 6.3 入力バリデーション

| チェック項目 | 実装 |
|------------|------|
| タスク名の空白チェック | `trim()` 後の空文字チェック |
| 日付の必須チェック | 追加フォーム・完了フォーム両方で実施 |

### 6.4 現バージョンの制約

| 項目 | 状況 | 対応方針 |
|------|------|---------|
| 認証・認可 | 未実装 | マルチユーザー対応時に実装 |
| localStorageの暗号化 | 未実装 | 機密データ保存時に実装 |
| セッション管理 | 未実装 | マルチユーザー対応時に実装 |

## 7. 非機能要件
